詳解

主に[楽譜管理]のリポジトリクラスで、検索メソッドの分岐と実行を行うメソッドのテスト を行うクラス

  • このテストで対象となるクラスは、[Musical_Score_Repo]である。
  • このテストに使用するデータベースは、本番環境と同一の種類の物を使用する。
  • 並列処理が可能なメソッドのテストの為、並列処理用の設定ファイルをインポートし、並列処理の 使用を許可しておく。

◆ configStaticSet()

void com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Musical_Score_Repo_Test.Musical_Score_Repo_FindAllBranch_Test.configStaticSet ( )
private

DIした環境変数を、静的変数でも使えるようにする。

  • 設けた理由としては、テスト後のデータベースのリセット確認の際に環境変数を用いる必要があるが、AfterAll は静的メソッドでしか実装できない為、通常のフィールド変数にDIされている環境変数が使えない。
  • そのため、新たに静的フィールド変数を作成して、その中にDIした環境変数の参照を入れておくことで、テスト後の データベースリセット確認にも環境変数を使用できるようにしている。
使用アノテーション
  • @PostConstruct
覚え書き
このメソッドは、セットアップのための非公開メソッドであり、外部に出さない。

◆ 検索分岐メソッドにおいて検索種別が全検索の際のテスト()

void com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Musical_Score_Repo_Test.Musical_Score_Repo_FindAllBranch_Test.検索分岐メソッドにおいて検索種別が全検索の際のテスト ( ) throws InterruptedException, ExecutionException, ParseException

リポジトリメソッド[findAllBranch]において、検索種別を[All]にした時の検索結果を判定する。

  • テストの内容としては、検索ワードや検索種別を指定して検索をした場合、想定された検索結果と、 テスト対象メソッドから返ってきた値が、順番含めて一致することを確認する。
  • また、本来Nullが許容されない引数に対してNullを渡した際に、エラーとなることを確認する。
大まかな処理の流れ
  1. テストケースから比較用のエンティティリストを取得し、そのリストをオフセット値やリミット、検索ワードで 加工し、検索条件に応じた比較用のエンティティリストを作成する。
  2. テスト対象メソッドを実行し、比較用のエンティティリストで比較を行う。
  3. その結果、想定通りの結果件数や、結果の順番、結果の内容であることを確認する。
  4. また、本来Nullが許容されない引数に対してNullを渡した際に、エラーとなることを確認する。
