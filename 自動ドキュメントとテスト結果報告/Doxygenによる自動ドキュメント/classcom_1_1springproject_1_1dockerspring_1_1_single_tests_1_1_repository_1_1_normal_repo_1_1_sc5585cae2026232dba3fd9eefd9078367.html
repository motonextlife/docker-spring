◆ Update_Success()

com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Score_Pdf_Repo_Test.Score_Pdf_Repo_Update_Test.Update_Success.Update_Success ( ) throws IOException

このインナークラス内で使用するフィールド変数のセットアップと同時に、参照元のテーブルへの データのセットアップを行う。

  • このテスト対象のテーブルは、他のテーブルを参照しているため、更新処理を行う場合は参照元のテーブルに あらかじめ参照可能なデータを入れておく必要がある。
  • なお、更新処理の為、テスト対象テーブルにも更新対象のデータがあらかじめ必要になる。
大まかな処理の流れ
  1. 使用するフィールド変数をセットする。
  2. 参照元のテーブルに参照可能なデータをあらかじめセットアップする。
  3. テスト対象テーブルに更新対象データをあらかじめセットアップする。

◆ 更新の成功ケース()

void com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Score_Pdf_Repo_Test.Score_Pdf_Repo_Update_Test.Update_Success.更新の成功ケース ( )

データの更新時の動作の確認を行う。なお、これは成功ケースである。

  • テストの内容としては、テストケースを保存した後、そのデータのシリアルナンバーで再度検索してデータを 取得。そのデータがテストケースと同一で、正常に更新されていることを確認する。
  • また、まだ更新していないデータに関しては、更新前と変わらない状態であることを確認する。
大まかな処理の流れ
  1. テストケースを保存した後、そのデータのシリアルナンバーで再度検索してデータを取得。
  2. テストケースと取得したデータを比較する。
  3. 更新したデータ以外を、更新前データのテストケースと比較し、一致することを確認する。
  4. 限界文字数まで敷き詰めたデータを、保存済みの既存のデータの適当なシリアルナンバーで更新し、そのシリアル ナンバーで再度検索した後、更新が成功かつ、データの内容が一致していることを確認する。
