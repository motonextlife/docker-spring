詳解

主に[権限管理]機能のテストにおいて、データベースとの通信テストの際に、あらかじめ データベース内にエラーとなる初期値データを格納してセットアップするクラス。

  • このクラスでは、データベースの初期化のほか、初期化時に使用したデータをテスト時の比較用として モックのエンティティに格納して返却が可能。
  • 必要に応じてファイルサーバーと連携してデータベースとファイルサーバーのどちらも初期化すること が可能である。
使用アノテーション

使用するテストケースクラスのDIのほか、YMLファイルからデータを読み取って専用エンティティへの マッピングを行う。

  • データベースやファイルサーバーのセットアップに必要なクラスを、インジェクションする。
  • データベースに格納するテストケースが記述されたYMLファイルを読み込んで、専用のエンティティに マッピングする。
引数
[in]config_factoryテスト設定ファイルの総合クラス
処理の大まかな流れ
  1. 引数で渡されたクラスを、コンストラクタインジェクションする。
  2. 指定されたYMLファイルのテストケースを読み込む。
  3. 読み込んだYMLファイルを専用のエンティティにマッピングする。
  4. インジェクションしたクラスを、フィールド変数に格納する。

◆ failDatabaseSetup()

Map< Integer, String > com.springproject.dockerspring.CommonTestCaseMaker.UsageAuthority.FailDatabase.Usage_Authority_Fail_Database_TestCase.failDatabaseSetup ( )

リポジトリの検査時に用いるデータベースに、エラーテストケースのデータを保存する。

  • なおセットアップ後は、データ検索に必要なシリアルナンバーと管理番号を返す。
  • ただし、データベースに格納できないようなエラーケース（例、列挙型や日付型）は対象外。
戻り値
データ検索に必要なシリアルナンバーと管理番号のマップリスト
処理の大まかな流れ
  1. 保存前に保存対象のテーブルを初期化するクエリを組み立てる。
  2. 組み上げたクエリを実行する。
  3. クエリ組み上げの過程で保存しておいた、データ検索に必要なシリアルナンバーと管理番号を戻り値とする。

List< Map< Usage_Authority_TestKeys, String > > com.springproject.dockerspring.CommonTestCaseMaker.UsageAuthority.FailDatabase.Usage_Authority_Fail_Database_TestCase.getDatabaseFail ( )
private

エラーテストケースを全て格納したリストを作成する。

  • テストケースに登録されている全てのエラーケースをリスト化したものを作成する。
  • このテストケースは、参照元のテーブルがエラーテストケースを格納した状態に初期化されている事が条件の元、 使用可能。参照制約があるため、その条件を満たせないとテストできない。
  • ただし、データベースに格納できないようなエラーケース（例、列挙型や日付型）は対象外。 これは、データベースに格納する前に制約に違反し保存できない為、テーブル内にテストケースを用意できないから。
戻り値
エラーケースのマップが順番に格納されたリスト
処理の大まかな流れ
  1. マッピングしたエンティティから、エラーテストケースのみを取り出す。
  2. 権限番号が参照元テーブルの権限番号を参照できるように、「権限番号がエラーのケース」以外のエラーケースに 関しては、確実に参照できてエラーが起こらない権限番号に上書きする。
  3. 取り出したデータは、順番にリストに格納する。
  4. 各項目の格納が終わったリストを全て結合し、一つのリストにしたものを戻り値とする。

項目[権限番号]でエラーになるテストケースを格納したマップリストを作成する。

  • マッピングしたエンティティから該当項目のエラーケースを取り出し、ベースとなる合格ケースのリストを 上書きすることで作成する。
  • 一つの項目のみがエラーとなるようにしているので、テストエラー時の原因を把握しやすくなる。
  • ただし、データベースに格納できないようなエラーケース（例、列挙型や日付型）は対象外。 これは、データベースに格納する前に制約に違反し保存できない為、テーブル内にテストケースを用意できないから。
戻り値
エラーケースのマップが順番に格納されたリスト
処理の大まかな流れ
  1. あらかじめ定義した配列内の文字を連結して、データを取得する際のキー名を組む。
  2. マッピングしたエンティティから、エラーテストケースのみを取り出す。
  3. あらかじめ作成した合格ケースのリストを取得し、そのリストの該当項目をエラーケースで上書きする。
  4. 上書きしたデータは、順番に新規のリストに格納する。
  5. 格納が終わったリストを戻り値とする。
