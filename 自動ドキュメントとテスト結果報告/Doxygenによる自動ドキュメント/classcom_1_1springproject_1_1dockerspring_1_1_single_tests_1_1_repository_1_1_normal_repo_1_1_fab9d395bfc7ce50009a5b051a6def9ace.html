com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Facility_Photo_Repo_Test.Facility_Photo_Repo_Update_Test.Update_Failed.Update_Failed ( ) throws IOException

このインナークラス内で使用するフィールド変数のセットアップと同時に、参照元のテーブルへの データのセットアップを行う。

  • このテスト対象のテーブルは、他のテーブルを参照しているため、更新処理を行う場合は参照元のテーブルに あらかじめ参照可能なデータを入れておく必要がある。
  • なお、更新処理の為、テスト対象テーブルにも更新対象のデータがあらかじめ必要になる。
大まかな処理の流れ
  1. 使用するフィールド変数をセットする。
  2. 参照元のテーブルに参照可能なデータをあらかじめセットアップする。
  3. テスト対象テーブルに更新対象データをあらかじめセットアップする。

void com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Facility_Photo_Repo_Test.Facility_Photo_Repo_Update_Test.Update_Failed.更新の失敗ケース ( )

データの更新時の動作の確認を行う。なお、これは失敗ケースである。

  • テストの内容としては、データベースのカラムの制約に違反し保存に失敗するデータを保存した際に、例外が 投げられエラーになることを確認する。
大まかな処理の流れ
  1. 失敗テストケースからデータを取得し、保存時にエラーになることを確認する。
  2. この一連の処理をテストケースの数だけ繰り返す。
