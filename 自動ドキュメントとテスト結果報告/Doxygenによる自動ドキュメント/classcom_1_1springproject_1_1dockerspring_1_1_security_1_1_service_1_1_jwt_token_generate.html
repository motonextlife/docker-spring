◆ decipher()

AccountDetails com.springproject.dockerspring.Security.Service.JwtTokenGenerate.decipher ( String  token) throws JWTVerificationException, Exception

引数で渡されたトークンを解析し、抽出したデータを認証情報として作成する。

  • トークンの解析の際は、公開鍵署名[ECDSA256]で必ず解析することを指定する。これによって、 署名方式の改ざんによる不正を防ぐ。
引数
[in]token解析対象のトークン
大まかな処理の流れ
  1. トークンの頭文字に付加されている[Bearer]を取り除く。
  2. 公開鍵と秘密鍵を用意する。
  3. 厳密に解析用の鍵を指定して、トークンの解析を行う。改ざんがあった場合はエラーを投げる。
  4. 解析が成功すれば、抽出したデータから、ユーザー名と権限情報リストを取得する。
  5. 取得した2つの情報で、認証情報用の専用エンティティを作成し、戻り値とする。

◆ makeToken()

String com.springproject.dockerspring.Security.Service.JwtTokenGenerate.makeToken ( AccountDetails  user_details) throws Exception

JWTトークンを、引数で渡された認証情報を用いて作成する。

  • トークンの有効期限は、環境変数の値に従うが、かなり短時間に設定されている。これによって、 ログアウト後のトークンのブラックリストを作成する必要がなく。ステートレス性が確保されるが、 定期的にフロント側からリクエストを行って、トークンの再発行を行わないといけない。
  • 署名を行う鍵に関しては、公開鍵方式を使う。これによって、共通鍵方式よりもブルートフォース 攻撃に強くすることが出来る。
引数
[in]user_detailsトークンに使用する認証情報
大まかな処理の流れ
  1. トークンに以下の情報を保存する。
    • ユーザー名
    • トークンの有効期間開始日（現在日時）
    • トークンの有効期間終了日（現在日時 + 環境変数の時間）
    • 権限情報リスト
  2. 公開鍵と秘密鍵を用意する。
  3. 保存した先ほどの情報を、公開鍵アルゴリズム[ECDSA256]で署名する。
  4. 生成したトークンの頭文字に[Bearer]と付加して、戻り値とする。

◆ tokenKeygen()

byte[] com.springproject.dockerspring.Security.Service.JwtTokenGenerate.tokenKeygen ( Boolean  private_key) throws Exception
private

トークンの解析や作成に必要な鍵のペアを作成する。

  • 作成する鍵の方式は公開鍵方式とする。鍵の種類は[EC/SHA1PRNG/256バイト]とする。
  • 鍵が既にファイル内に存在する場合はそちらを用いて、ない場合は新規に作成して用いる。
引数
[in]private_key「True」で作成した秘密鍵、「False」で作成した公開鍵を取得。
大まかな処理の流れ
  1. 鍵のペアを作成するファイルインスタンスを作成する。なお、鍵を作成するディレクトリは 簡単にはアクセスできない場所にする。少なくともクラスパス上には作らない。
  2. 二つのファイルが既に存在すれば、そのファイルを読み取り戻り値とする。
  3. 二つのファイルの片方でもない場合は、すべての鍵ファイルを削除し、新たに鍵のペアを 作成する。
