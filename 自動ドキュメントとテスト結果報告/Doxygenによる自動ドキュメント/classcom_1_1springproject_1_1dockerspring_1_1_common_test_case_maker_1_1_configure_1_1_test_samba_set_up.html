詳解

テストコード全体において、ファイルサーバーを使用可能なように接続情報などを定義して、 初期化をするクラスを格納するファイル。

  • ファイルサーバーへのアクセスのための接続情報を、インスタンスの作成の際に定義する。
  • 前述の接続情報を元に、ファイルサーバーへの初期化データの挿入を行う。
  • テストが終わった後は、使用したデータの削除を行い、ファイルサーバーをリセットする。
  • なお、このクラスは通常データ用と履歴データ用で共用で用いる。

com.springproject.dockerspring.CommonTestCaseMaker.Configure.TestSambaSetUp.TestSambaSetUp ( TestDataSource  data_source) throws CIFSException

ファイルサーバーへの接続情報の定義を行う。

  • 環境変数に指定されたURLやバージョン情報をもとに定義する。
  • また、ファイルサーバー内の基本となるファイルパスを、環境変数から取得し定義する。
引数
[in]data_sourceテスト用の環境変数設定ファイル。
処理の大まかな流れ
  1. 接続情報のバージョン情報に、ファイルサーバーの最小と最大のバージョン番号を設定。
  2. 基本コンテキストを作成し、ファイルサーバーへのアクセスのための、ユーザー名とパスワードを登録する。
  3. 作成した認証情報を元に接続コンテキストを作成し、フィールド変数に格納。
  4. 環境変数からファイルサーバー上のファイルパスを取得し、フィールド変数に格納。

String com.springproject.dockerspring.CommonTestCaseMaker.Configure.TestSambaSetUp.sambaHistoryOutput ( String  read_file_name,
String  output_file_path 
) throws IOException

履歴データのファイルサーバーへの初期値データの挿入を行う。

  • 指定されたテストケース内のファイルを読み取って、ファイルサーバーに格納する。
  • 格納するデータは、GZIP圧縮して保存する。
  • 格納するデータは、圧縮前にハッシュ値を計算し、戻り値とする。
引数
[in]read_file_nameテストケースから読み取るデータのファイル名
[in]output_file_pathファイルサーバー上の書き込み対象のファイル名。
処理の大まかな流れ
  1. テストケースから読み取るバイナリデータのファイル名を指定し、入力ストリームを開く。
  2. あらかじめ作成された接続情報と、接続先のファイルサーバー上のファイル名を指定して、出力ストリームを開く。
  3. データの格納の前に、読み取ったデータのハッシュ値を計算し、保存しておく。
  4. 読み取ったテストケースのデータを、ファイルサーバーに「GZIP圧縮しながら」順次書き込んでいく。
  5. 保存しておいたハッシュ値を戻り値とする。
