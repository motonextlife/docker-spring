詳解

データの更新処理に関して、一意制約違反によって失敗するテストを実施するインナークラス。

なお、一意制約対象の値は、管理番号とユーザー名である。

com.springproject.dockerspring.SingleTests.Repository.NormalRepo.System_User_Repo_Test.System_User_Repo_Update_Test.Unique_Failed.Unique_Failed ( ) throws IOException

このインナークラス内で使用するフィールド変数のセットアップと同時に、テスト対象テーブルへの 初期データを準備する。

  • 一意制約のテストを行うため、あらかじめ内容が被るデータを入れておく必要がある。
  • また、一意制約テストの際に、更新を除外するデータのシリアルナンバーを控えておく。
  • このテスト対象のテーブルは、他のテーブルを参照しているため、更新処理を行う場合は参照元のテーブルに あらかじめ参照可能なデータを入れておく必要がある。
大まかな処理の流れ
  1. 使用するフィールド変数をセットする。
  2. テスト対象テーブルへの初期データを準備する。
  3. 更新を除外するデータのシリアルナンバー（一意制約エラーの管理番号と同じ物）を控える。

void com.springproject.dockerspring.SingleTests.Repository.NormalRepo.System_User_Repo_Test.System_User_Repo_Update_Test.Unique_Failed.更新の一意制約違反のケース ( )

データの更新を行った際に、一意制約違反によってエラーが出ることを確認する。

  • テストの内容としては、データベースのカラムの一意制約に違反し保存に失敗するデータを保存した際に、例外が 投げられエラーになることを確認する。
大まかな処理の流れ
  1. 失敗テストケースからデータを取得し、保存時にエラーになることを確認する。
  2. なお、エラー確認をする際には、一意制約違反として投入する管理番号やユーザー名に紐付いている シリアルナンバー「以外」のデータに対して行う。
