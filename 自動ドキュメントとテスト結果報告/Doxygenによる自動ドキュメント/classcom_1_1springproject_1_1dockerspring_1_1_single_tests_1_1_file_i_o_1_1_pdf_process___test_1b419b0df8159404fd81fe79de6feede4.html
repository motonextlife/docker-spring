主に[楽譜管理]のPDFデータの出力機能のテストを行うクラス

  • このテストで対象となるクラスは、[Musical_Score_PdfImpl]である。
  • 主に、PDFデータが正常に出力されるかどうかのテストを行う。
  • デザインに関しては、このテストでは検査できないので、出力した後に目視で確認を行うしかない。

void com.springproject.dockerspring.SingleTests.FileIO.PdfProcess_Test.Musical_Score_PdfImpl_MakePdf_Test.PDF出力機能の動作確認と生成ファイルの存在確認 ( ) throws IOException, ParseException

PDFデータの出力テストを行う。

  • 検査方法としては、テスト対象クラスを実行した後に、PDFデータが存在するかを確認する。
  • 本来であれば、PDFデータの同一性チェックを行うべきだが、PDFのデザインは今後頻繁に変わる不定値の 為、jUnitでの自動テストが困難。
  • そのため、自動テストでは生成ファイルの存在確認のみにとどめる。デザインは目視で確認を行う事。
大まかな処理の流れ
  1. テスト対象クラスを実行し、返ってきた一時ファイルの存在確認。
  2. テスト対象クラスにNullを渡して、エラーが発生することを確認。
  3. テスト後は一時ファイルを削除する。
