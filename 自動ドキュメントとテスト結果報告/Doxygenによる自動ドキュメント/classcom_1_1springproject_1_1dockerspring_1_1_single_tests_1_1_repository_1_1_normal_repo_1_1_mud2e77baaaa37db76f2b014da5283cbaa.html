private

DIした環境変数を、静的変数でも使えるようにする。

  • 設けた理由としては、テスト後のデータベースのリセット確認の際に環境変数を用いる必要があるが、AfterAll は静的メソッドでしか実装できない為、通常のフィールド変数にDIされている環境変数が使えない。
  • そのため、新たに静的フィールド変数を作成して、その中にDIした環境変数の参照を入れておくことで、テスト後の データベースリセット確認にも環境変数を使用できるようにしている。

◆ 対象の楽譜番号のデータを個別取得する際のテスト()

void com.springproject.dockerspring.SingleTests.Repository.NormalRepo.Musical_Score_Repo_Test.Musical_Score_Repo_Search_Test.対象の楽譜番号のデータを個別取得する際のテスト ( )

リポジトリメソッド[findByScore_Id]を実行した際の検索結果を判定する。

  • テストの内容としては、管理番号を完全一致指定して検索をした場合、想定された検索結果と、テスト対象 メソッドから返ってきた値が一致することを確認する。
  • 検索ワードにNullを渡した際には、検索結果が0件となることを確認する。
大まかな処理の流れ
  1. テストケースから比較用のエンティティリストを取得し、比較対象のデータの管理番号を取得する。
  2. 取得した管理番号を渡してテスト対象メソッドを実行し、比較用のエンティティリストで比較を行う。
  3. その結果、想定通りの結果の内容であることを確認する。
  4. 検索ワードにNullを渡した際には、検索結果が0件となることを確認する。
